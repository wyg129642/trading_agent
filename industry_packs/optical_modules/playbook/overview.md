# 光通信 / 光模块行业建模总览

## 产业结构要点

光模块产业链从上游到下游：**激光芯片 (EML/CW) → 光器件 → 光模块 → 交换机/服务器集成**。

核心终端需求：**AI 训练集群、云数据中心互联、数通客户 (Google/MSFT/AWS/Meta/Oracle)、
电信 (传统/5G)**。当前数通+AI 占比显著上升。

## 建模首要逻辑

1. **业务部门拆分优先级**：
   - `400G / 800G / 1.6T 模块` — 必须分别独立拆量价（技术代际差异大）
   - `EML / CW 芯片`，进一步细分 `100G / 200G` 等速率（同类产品不同代际 ASP 差数倍）
   - `OCS / CPO / DCI` — 管理层多直接给收入，不必做量价分拆
   - `传统通信 / 工业` — 稳定/缓降，按 `-5% 到 +5%` 填数或按管理层指引

2. **量的铆定**（按优先级）：
   - 管理层指引（业绩会/Investor Day）
   - 专家口径（久谦、Alpha派、Third Bridge 访谈）— **至少 2 位专家，且需要对口径差异做解读**
   - AI 基建铆定（Google TPU 出货、NVIDIA GPU 季度发货、大客户数据中心投资节奏）
   - 产能约束（如泰国厂、FAB 产能上限）

3. **价的判断**（按优先级）：
   - 管理层指引框架
   - "保价保供" 合同（大客户提前锁价）
   - 专家对涨价/降价节奏判断
   - 历史代际 ASP 年降 10-20% 经验
   - **纯涨价 vs 增量的分歧**：如果是纯涨价，则收入增量 ≈ 毛利增量，营业利润率显著提升

4. **毛利率参考**：
   - 100G EML GM ~60%；200G EML GM ~50%；CW GM ~30% (考虑良率)
   - 模块成品 GM 通常 25-35%；AI 关键产品 400G+ 可能 35-45%

## 常见陷阱

- **同一指标不同专家口径差别大**（如 EML+CW 出货：久谦 15,000 万颗 vs 自己 12,000 万颗）。
  **处理方式**：两个数字都保留在 `alternative_values`，主值选取原则：
  - 口径包含更广泛产品者取大；
  - 近期业绩会佐证者取之；
  - 若专家为核心客户供应链，权重更高。
- **ASP 取年末 vs 年均 vs 某季**：模块行业建议用**全年均价**，芯片行业看产品代际。
- **季度/年度混用**：注意区分管理层说的 "quarterly 25/26/27 出货" 是季均还是全年。
